# 基于液态神经网络 (LNN) 的沙门氏菌基因组预测 — 实验计划

## Context

**目标**：利用液态神经网络（Liquid Neural Network）架构，根据细菌基因序列预测该菌是否为沙门氏菌（*Salmonella*），并进一步区分血清型。

**数据规模**：
| 数据源 | 类别 | 原始基因组 | Chunks 数 |
|--------|------|-----------|-----------|
| `zheng-yangpin/` | 阳性（沙门氏菌） | 354 个 .fna | — |
| `fu-yangpin/` | 阴性（非沙门氏菌） | 161 个 .fna | — |
| `serotype_data/*/` (6 种血清型) | 阳性（已知血清型） | 300 个 .fna.gz | ~298,628 |
| `negative_species/*/` (6 种菌) | 阴性（已知种属） | 300 个 .fna.gz | ~123,154 |

**Chunk 格式**：每条约 1000-2000 bp 的 DNA 片段，标签 CSV 包含 file_path / label / species / group / difficulty

**核心挑战**：
- DNA 长序列依赖建模（基因组长度可达数百万 bp）
- 跨物种泛化（训练中未见过的血清型/物种）
- 小参数量下的高效推理

---

## Phase 1: 环境搭建与数据准备（第 1–2 天）

### 1.1 开发环境
- Python 3.10+ / PyTorch 2.x
- 关键库：`biopython`, `scikit-learn`, `pandas`, `numpy`, `matplotlib`, `seaborn`
- LNN 框架选择：**ncps-torch**（PyTorch 版 CfC/LTC）或手动实现 CfC 细胞
- GPU：单卡 NVIDIA（RTX 3060 级别即可，LNN 参数量极小）

### 1.2 数据加载与编码

**方案 A（推荐起步）：One-hot 编码**
- 将 A/T/C/G 映射为 4 维向量：`A=[1,0,0,0], T=[0,1,0,0], C=[0,0,1,0], G=[0,0,0,1]`
- 固定序列长度（如 1024 bp），超长截断，不足补齐
- 输入形状：`(batch, seq_len=1024, channels=4)`

**方案 B（进阶）：k-mer 频率向量**
- k=3~6，每个 chunk 转换为 k-mer 频率向量
- 输入形状：`(batch, 4^k)`，如 k=4 → 256 维

**方案 C（可选探索）：可学习 Embedding**
- 将每个 k-mer 视为 token，学习 dense embedding
- 需要更多计算资源

### 1.3 数据集划分
```
Task 1 — 二分类（是否沙门氏菌）:
  阳性: serotype_data/**/chunks + zheng-yangpin chunks  ≈ 298k
  阴性: negative_species/**/chunks + fu-yangpin chunks  ≈ 123k
  划分: 按基因组来源 split（避免同一基因组同时出现在 train/test）
  Train/Val/Test = 70% / 15% / 15%

Task 2 — 血清型多分类（仅沙门氏菌）:
  类别: dublin, enteritidis, heidelberg, infantis, newport, typhimurium
  数据: serotype_data/**/chunks ≈ 298k
  划分: 同上，按基因组来源 split
```

### 1.4 类别不平衡处理
- 阳性样本约 2.4 倍于阴性 → 使用 **weighted random sampler** 或 **focal loss**
- 多分类中各血清型也不均衡（heidelberg 94k vs typhimurium 13k）→ 考虑过采样/欠采样

## Phase 2: 基线模型（第 3–5 天）

在构建 LNN 之前，先建立基线以衡量 LNN 的相对表现。

### 2.1 传统机器学习基线
- **k-mer + XGBoost**：k=4 频率向量 + XGBoost 分类器
- **k-mer + Random Forest**：同上，对比

### 2.2 深度学习基线
- **1D-CNN**：3 层卷积 + Global Max Pooling + FC
- **BiLSTM + Attention**：2 层 BiLSTM + 自注意力池化
- **Transformer Encoder**（小型）：4 层、4 头，作为"标准架构"对照

### 2.3 基线评估指标
- Accuracy, Precision, Recall, F1-score, AUC-ROC
- **关键指标**：按基因组来源分组的交叉验证准确率（评估泛化到新基因组的能力）

## Phase 3: LNN 模型设计与实现（第 5–8 天）

### 3.1 核心 CfC (Closed-form Continuous-time) 细胞

LNN 的核心是连续时间动力学：

$$\dot{x} = -x/\tau + f(x, I, \theta)$$

CfC 对此 ODE 提供了闭式近似，无需数值求解器，训练高效。

实现方案：
1. **使用 ncps-torch 库**：`from ncps.torch import CfC`（推荐起步）
2. **手动实现**：参考 Hasani et al. 2022 的 CfC 公式，从头用 PyTorch 实现以完全掌控

### 3.2 架构设计

```
输入: (batch, seq_len, 4)   # one-hot DNA
  ↓
Positional Encoding (可选: 可学习 PE 或 RoPE)
  ↓
LNN Backbone (CfC Stack):
  ├─ CfC Layer 1: hidden=128, 感受野逐时间步处理
  ├─ LayerNorm
  ├─ CfC Layer 2: hidden=64
  ├─ LayerNorm
  └─ CfC Layer 3: hidden=32
  ↓
序列池化: 最后时间步输出 或 注意力池化
  ↓
分类头: Linear(32, num_classes) + Sigmoid/Softmax
```

**参数量估算**：
- CfC 每层约 1-5k 参数（vs LSTM 的 50k+，Transformer 的 100k+）
- 整个 LNN 模型约 **20k-50k 参数**

### 3.3 关键设计选择

| 维度 | 选项 A | 选项 B | 建议 |
|------|--------|--------|------|
| LNN 细胞类型 | LTC (ODE solver) | CfC (闭式解) | **CfC** — 训练快 5-10x |
| 时间步处理 | 逐碱基滑动 | 分块并行 | **逐碱基** — 保留连续特性 |
| 序列长度 | 512 bp | 1024/2048 bp | **1024** — 平衡上下文与效率 |
| 位置编码 | 无 | 可学习 PE | **可学习 PE** — 测试是否需要 |

### 3.4 LNN 的三个变体（用于消融实验）
1. **LNN-Small**：单层 CfC(64) → 约 5k 参数
2. **LNN-Medium**：3 层 CfC(128→64→32) → 约 25k 参数
3. **LNN-Large**：3 层 CfC(256→128→64) → 约 80k 参数

## Phase 4: 训练策略（第 8–10 天）

### 4.1 训练配置
```yaml
优化器: AdamW
学习率: 1e-3, warmup 500 steps → cosine decay
Batch size: 256 (LNN 参数量小，可增大)
Epochs: 100 (early stopping patience=15)
Loss: Binary Cross Entropy (二分类) / Cross Entropy (多分类)
正则化: Dropout(0.1), Weight Decay(1e-4)
```

### 4.2 训练流程
1. 先训二分类（Task 1），快速验证 LNN 有效性
2. 再训多分类（Task 2），评估细粒度区分能力
3. 每个任务跑 3 次取均值 ± 标准差

### 4.3 关键技巧
- **梯度裁剪**：CfC 的 ODE 本质可能导致梯度问题，clip norm=1.0
- **混合精度训练**：AMP 加速
- **Warmup**：前 5% steps 线性 warmup，防止初期不稳定

## Phase 5: 评估与对比分析（第 10–12 天）

### 5.1 主要评估维度

**A. 标准分类性能**
| 模型 | Accuracy | F1 | AUC | 参数量 | 推理时间(ms) |
|------|----------|-----|-----|--------|-------------|
| k-mer + XGBoost | - | - | - | - | - |
| 1D-CNN | - | - | - | - | - |
| BiLSTM | - | - | - | - | - |
| Transformer | - | - | - | - | - |
| **LNN-Small** | - | - | - | ~5k | - |
| **LNN-Medium** | - | - | - | ~25k | - |
| **LNN-Large** | - | - | - | ~80k | - |

**B. 分布外 (OOD) 泛化**
- 训练集留出一种血清型（如 typhimurium），测试 LNN 在该血清型上的识别能力
- 训练集留出一种阴性物种（如 pseudomonas），测试泛化

**C. 序列长度鲁棒性**
- 测试不同输入长度（256/512/1024/2048 bp）下的性能变化

**D. 数据效率**
- 只用 10%/25%/50% 训练数据，对比各模型的性能衰减
- LNN 的核心卖点之一是小数据学习能力

### 5.2 统计显著性
- 每个实验重复 3 次（不同随机种子）
- 报告均值 ± 标准差
- 必要时做配对 t-test

## Phase 6: 消融实验与可解释性（第 12–14 天）

### 6.1 消融实验
1. **CfC vs LTC vs 简单 RNN**：对比 ODE 动力学是否必要
2. **有无位置编码**：DNA 序列是否真的需要位置信息
3. **序列处理方向**：单向前向 vs 双向
4. **池化策略**：最后步 vs 注意力池化 vs 均值池化
5. **输入编码**：one-hot vs k-mer embedding vs 可学习 embedding

### 6.2 可解释性分析
- **时间步重要性**：分析 LNN 在处理序列时，哪些位置（碱基）对决策权重最大
- **隐藏状态轨迹可视化**：用 PCA/t-SNE 降维 CfC 的隐藏状态，观察不同菌种的动力学轨迹
- **与已知毒力基因/血清型决定因子的对照**：检查 LNN 关注的区域是否对应已知的生物学特征（如 *fliC*, *rfb* 基因簇）

## 技术风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|------|---------|
| CfC 在长序列上训练不稳定 | 中 | 先用短序列 (512bp) 验证，逐步加长；梯度裁剪 |
| LNN 效果不如基线 | 低-中 | 从消融实验找原因，调整序列编码策略 |
| 数据量过大，训练慢 | 低 | CfC 本身参数少效率高；使用 DataLoader 多进程 |
| 过拟合到特定基因组 | 中 | 严格按基因组来源划分 train/test；做 OOD 测试 |
| ncps-torch 兼容性问题 | 中 | 手动实现 CfC 细胞作为后备 |

## 验证计划

1. **单元测试**：DNA encoding 正确性（已知序列 → 已知 one-hot 矩阵）
2. **过拟合测试**：在 10 个样本上训练至 100% 准确率，验证模型容量
3. **梯度检查**：CfC 细胞反向传播无 NaN
4. **端到端测试**：从原始 .fna 文件到最终预测的完整 pipeline
5. **外部验证**：下载 5-10 个不在训练集中但已知血清型的公共基因组，测试预测准确率
